При входе на стартовую страницу приложения - открывается ЭФО, в левой части которого находится информация о пользователе и кнопка, позволяющая при необходимости сменть пользователя, а также вход для администратора Чек-листа (административная панель)

 Центральная часть ЭФО содержит набор вкладок с информационным полем:
–	вкладка "Новая проверка" – позволяет создавать новые черновики Чек-листа путем нажатия на кнопку «Начать заполнение». При этом, если за указанную дату уже есть черновик или завершенный Чек-лист по указанному ВСП, то никто в этом ВСП не может больше начать заполнение чек листа повторно.На данной вкладке должно быть указано ФИО авторизованного пользователя, его логин, ВСП сотрудника. Авторизация пользователя и определение ВСП сотрудника должно происходить автоматически. Дата всегда равна дате текущего дня и недоступна для корректировки пользователю.Предусмотреть также на вкладке «Новая проверка» возможность изменения ВСП пользователя. 

–	вкладка «История проверок» – хранит информацию по обработке и завершению авторизованным пользователем Чек-листа в табличной форме, имеющим набор полей: id сессии, дата проверки, филиал, ВСП, статус проверки, Дата и время начала, Дата и время заверешения, кол-во выполненных проверок, всего проверок.
Также на данной вкладке можно выбрать необходимую сессию и нажав на кнопку «Показать результаты» увидеть перечень выполненных контролей, статус завершения Чек-листа, дату и время завершения. А также на данной вкладке имеется возможность удаления черновика авторизованным пользователем, путем выбора ID сессии и нажатия на кнопку «Удалить»
 
–	вкладка «Нерабочие дни ВСП» – позволяет внести информацию по нерабочим дням ВСП и имеет 2 основных поля: Внесение нерабочих дней ВСП и Ранее добавленные нерабочие дни.
Для внесения информации по нерабочему дню ВСП на данной вкладке предусмотрено поле ВСП РФ авторизованного пользователя, выбор даты  нерабочего дня из календаря, выбор причины из выпадающего списка. Предусмотреть возможность при выборе из списка причины «Другое», вносить ручные корректировки для полноты описания причины. Заполненные данные сохранаются путем нажатия на кнопку «Добавить нерабочий день». 
В поле «Ранее добавленные нерабочие дни хранится информация о ранее добавленных нерабочих днях ВСП, которую можно увидеть путем простановки галочки в поле «Показать все выходные вашего ВСП». Для необходимости удаления выходного дня по ВСП, на вкладке необходимо выбрать ранее добавленный нерабочий день и нажать на кнопку «Удалить». 


–	вкладка «Аналитика по филиалу» – позволяет увидеть проверки по ВСП, управлять черновиками и содержит список всех проверок. Данная вкладка содержит 3 основных поля: Аналитика проверок вашего филиала, Управление черновиками, Список всех проверок.
В поле «Аналитика проверок вашего филиала» автоматически указывается филиал/ВСП авторизованного пользователя и имеется возможность выбора дат проверок за период из календаря.
В поле «Управление черновиками» имеется информационное поле с информацией по количеству черновиков ВСП авторизованного пользователя. Для удаления черновика любому пользователю одного ВСП необходимо выбрать черновик для удаления из выпадающего списка, проставить галочку в поле «Подтверждаю удаление выбранных черновиков» и нажать на кнопку «Удалить выбранные черновики» 
В поле «Список всех проверок» отражается перечень всех проверок по ВСП в табличной форме с набором полей: ID сессии, Сотрудник, Дата, ВСП, Статус, Начало, Заверешени, Выполнено



–	вкладка “Визуализация”  – отражает информацию по статусу заполнения чек-листов по ВСП и имеет 1 поле “Статус заполнения чек-листов по вашему филиалу”, в котором автоматически определяется и отражается ВСП авторизованного пользователя и имеется возможность выбора даты заполнения из календаря. При нажатии на кнопку “Показать статус” открывается форма с круговой диаграммой, в которой отражается “ Соотношение заполненных и незаполненных ВСП”, где зеленым цветом отражаются заполненные ВСП, а красным –не заполненные ВСП, в т.ч. Содержится % соотношение и  “Детализация по ВСП вашего филиала” в табличной форме, которая содержит поля: ВСП, Статус.
